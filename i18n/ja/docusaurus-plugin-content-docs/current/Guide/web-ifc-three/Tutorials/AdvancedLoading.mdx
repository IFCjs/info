## はじめに

🥶🥶🥶
しかし、非常に大きなモデルを読み込むと、アプリケーションが何秒かフリーズしてしまうという問題があります。この問題は、モデルから多くのプロパティを抽出したいときにも発生します。これを回避する方法はあるのでしょうか？

🔥
もちろん、IFC.js はこの状況を予見して、**マルチスレッド**を実装しています。つまり、重い操作はアプリケーションをブロックしない並列プロセスで実行され、プロセスが終了すると非同期に更新されるのです。

👌
これは複雑そうに聞こえますし、実際そうなのですが。しかし、心配はいりません。私たちは、数行のコードであなたの BIM アプリケーションにこの機能を持たせることができるよう、すべてセットアップしてあるのです。

  </a>
  でご覧いただけます。
</IfcCard>

## 方法

### ウェブワーカーをセットアップする

👩‍🏫
マルチスレッドを有効にするためには、**webworker**を設定する必要があります。しかし、ウェブワーカーとは何でしょうか？ウェブワーカーとは、ウェブブラウザがマルチスレッドで動作できるようにするための仕組みです。でも、聞いたことがなくても心配しないでください。**IFC.js でマルチスレッドを使うのに、ウェブワーカーは必要ありません**。

📦📦📦
`npm` や `yarn` などのパッケージマネージャを使用している場合は、 `node_modules` というフォルダがあり、そこにプロジェクトで使用しているライブラリのフォルダがすべて格納されているはずです。`node_modules/web-ifc-three` の中に、ウェブワーカーを使用するために必要な 2 つのファイルがあります。`IFCWorker.js` と `IFCWorker.js.map` です。

<IfcCard>
  この2つのファイルを、最終的なアプリケーションでサービスを提供するプロジェクト内のディレクトリにコピーする必要があります。これは、使用している技術（バニラJS、React、Angularなど）によって異なります。wasmファイルに使用しているのと同じディレクトリを使用することができます。
</IfcCard>

🌍
そして、このファイルがどこにあるのかを、wasm ファイルと同じように指定する必要があります。これは次のようにできます。

```js
async function setUpMultiThreading() {
  const manager = ifcLoader.ifcManager;
  // これらのパスは、プロジェクトの構成方法によって異なります。
  await manager.useWebWorkers(true, "IFCWorker.js");
  await manager.setWasmPath("../../../");
}

setUpMultiThreading();
```

🥶⛔
そして、voi lá! 信じられないかもしれませんが、あなたはすでにプロジェクトでマルチスレッドを有効にしています。これで、アプリケーションでファイルを読み込んだり、一度に多くのプロパティを取得しようとすると、**アプリケーションはフリーズしない**ことがわかります。

<IfcAlert>
  Webworkerを起動した場合、wasmファイルのパスはWebworkerからの相対パスである必要があることに注意してください。
</IfcAlert>

🤖❓📚
ウェブワーカーのバージョンは、使用しているライブラリ\*\*のバージョンと一致しなければならないことに留意してください。つまり、将来的に `web-ifc-three` のバージョンを更新した場合、これらのファイルを再度コピーする必要があります。`cpy` のようなファイルをコピーするコマンドラインライブラリを使用すれば、これを自動化することができます。

### ロードの進捗状況

🔥
BIM アプリケーションでマルチスレッドを使用すると、数多くの利点があります。最も直接的なのは、**モデルのロードの進捗状況をユーザーに表示できるようになったことです**。まず、進捗状況を表示するための簡単な HTML メッセージを追加します。

```html
div id="text-container">
<p>Progress:</p>
<p id="progress-text">0</p>
<p>%</p>
/div>
```

✨✨✨
次に、この HTML 要素をモデルロードイベントに**リンクさせます**。これは `setOnProgress()` メソッドで簡単に行うことができます。また、基本的な計算を適用して、**進捗をパーセンテージに変換してみましょう**。

```js
function setupProgressNotification() {
  const text = document.getElementById("progress-text");
  ifcLoader.ifcManager.setOnProgress((event) => {
    const percent = (event.loaded / event.total) * 100;
    const result = Math.trunc(percent);
    text.innerText = result.toString();
  });
}

setupProgressNotification();
```

🌈
すべて正しく行われた場合、以下のようなビューアが表示されるはずです。IFC モデルを読み込もうとすると、**読み込み処理が 3D ビューをブロックしない**だけでなく、**html テキストに読み込みの進捗がリアルタイムで表示される**ことがわかります。
