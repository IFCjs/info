## web-ifc-three でできるすごいことリスト

📃
すべての API はドキュメント化されていますので、このドキュメントで紹介されているオブジェクトやメソッドを使用すると、使用している IDE にかかわらず、Intellisense でヘルプが表示されるはずです。[チェックしてみてください！ ](https://github.com/IFCjs/web-ifc-three/blob/main/web-ifc-three/src/IFC/components/IFCManager.ts)

👩‍🏫
しかし、インテリセンスやコメントを読むことは、最も快適なことではないことを理解しているので、このページでは、API でできることの概要を説明します。すべての内容は、後ほど個別のチュートリアルで詳しく説明します。

## IfcLoader

💎
ライブラリーからインポートする唯一のオブジェクトです。IFC を扱うために必要なすべてのロジックが含まれています。その`load()`メソッドと`loadAsync()`メソッドを使用して、URL から IFC を読み込むことができます。<a>他の Three.js Loader </a>と同様です。例えば、IFC を読み込むには以下のようにします。

```js
import { IFCLoader } from "web-ifc-three/IFCLoader";

const ifcLoader = new IFCLoader();
ifcLoader.load("models/Example_model.ifc", (ifcModel) => scene.add(ifcModel));
```

🏠🏠🏠
`web-ifc-three`では、シーン内に複数のモデルを読み込むことができます。API の操作の多くは、指定されたモデルに対して実行されます。どのモデルを操作したいかを表すには、そのモデルの`ModelID`を指定する必要があります。

🔍
モデルの ID は、生成されたメッシュのプロパティ`modelID`で取得できます。

```js
const modelID = ifcModel.modelID;
```

✌
この API にアクセスするには 2 つの方法があります。

- `IfcLoader`が`load`と`loadAsync`で生成する`IfcModel`のインスタンスを介して。

- `IfcLoader.IfcManager`を介して。

<IfcAlert>
  <b>IfcModel</b>のインスタンスと<b>IfcLoader.IfcManager</b>
  の両方は、APIにアクセスするために互換的に使用することができます。
  <b>IfcModel</b>
  を使用する場合、ModelIDを引数として与える必要はありません(暗黙的に知られています)。
</IfcAlert>

## 🧰 セットアップ

### setWasmPath

```js
  async IfcLoader.IfcManager.setWasmPath (
                        path: string
                        ): void;
```

🧠
web-ifc.wasm`と`web-ifc-mt.wasm`のファイルの場所を指定します。これらのファイルは、web-ifcファイルであり、IFC.jsを使ってアプリケーションを作成する際に必要となります。これらのファイルは `node_modules/web-ifc/` で見つけることができます。

<IfcCard icon={"🚧"} title={"使う道具には気をつけよう"}>
  React、Angular、Vue、Svelteなどのフレームワークやライブラリを使用している場合、プロジェクトのルートパスと提供されるアプリケーションのルートパスが一致しないことがあります。静的に提供されるファイルのパスがどのように管理されているか、
  <a href="https://github.com/IFCjs/examples">各ケース</a>
  で確認する必要があります。
</IfcCard>

#### 関数の引数

- `path` `web-ifc.wasm`のルートです。

#### 例

`web-ifc.wasm` が dist/wasmDir にある場合。

```js
await ifcLoader.setWasmPath("dist/wasmDir/");
```

### setupThreeMeshBVH

```js
  IfcLoader.IfcManager.setupThreeMeshBVH (
                        computeBoundsTree: any,
                        disposeBoundsTree: any,
                        acceleratedRaycast: any
                        ): void;
```

⚡⚡⚡
この方法により、オブジェクトのピッキングが格段に速くなり、特に重いジオメトリを持つ非常に大きなモデルの場合に有効です。この方法により、IFC.js は Garrett Johnson 氏の素晴らしい[ライブラリ](https://github.com/gkjohnson/three-mesh-bvh)を使用することができます。`npm i three-mesh-bvh`または `yarn add three-mesh-bvh` でインストールできます。

<IfcCard>
  この方法は必須ではありませんが、中/大IFCモデルのオブジェクトを60fpsで選択できるようにしたい場合は必要です。
</IfcCard>

#### 関数の引数

- `computeBoundsTree` 新しい BVH を構築し、それを boundsTree に割り当て、新しいインデックスバッファをジオメトリに適用する、プリメイドの BufferGeometry 拡張関数です。- [ソース](https://github.com/gkjohnson/three-mesh-bvh#computeboundstree)。

- `disposeBoundsTree` BVH を破棄する BufferGeometry 拡張関数です。- [ソース](https://github.com/gkjohnson/three-mesh-bvh#disposeboundstree)を参照してください。

- `acceleratedRaycast` THREE.Mesh.raycast と同じシグネチャーを持つ、高速化されたレイキャスト関数です。BVH が利用可能な場合はレイキャスティングに使用し、そうでない場合はビルトインのアプローチに戻ります。- [ソース](https://github.com/gkjohnson/three-mesh-bvh#acceleratedraycast)を参照してください。

#### 例

```js
import { IFCLoader } from "web-ifc-three/dist/IFCLoader";

import {
  acceleratedRaycast,
  computeBoundsTree,
  disposeBoundsTree,
} from "three-mesh-bvh";

const ifcLoader = new IFCLoader();
ifcLoader.ifcManager.setupThreeMeshBVH(
  acceleratedRaycast,
  computeBoundsTree,
  disposeBoundsTree
);
```

### setOnProgress

```js
  IfcLoader.IfcManager.setOnProgress (
      onProgress: (event: ParserProgress) => void
                                    ): void;
```

⏳
IFC が 10％ロードされるごとに呼び出されるコールバック関数を設定します。これにより、ローディングバーを表示して、ユーザーにローディングの状況を示すことができます。

#### 関数の引数

- `onProgress`: 10%ごとに呼び出されるコールバック関数です。この関数は、2 つのプロパティを持つオブジェクトを受け取る 1 つの引数を持つ必要があります。`loaded`（読み込まれたオブジェクトの数）と `total`（ファイル内のオブジェクトの数）です。

#### 例

```js
function exampleCallback(event) {
  const progress = (event.total / event.progress) * 100;
  console.log("Progress: ", progress, "%");
}

ifcLoader.ifcManager.setOnProgress(exampleCallback);
```

### applyWebIfcConfig

```js
  async IfcLoader.IfcManager.applyWebIfcConfig (
                                    settings: LoaderSettings
                                    ): void;
```

🎹🎹🎹
このライブラリが内部で使用しているパーサーである[web-ifc](https://ifcjs.github.io/info/ja/docs/Guide/web-ifc/Introduction)の設定を適用します。

#### 関数の引数

- `settings`: 以下のフィールドを含むオブジェクトです。

- `COORDINATE_TO_ORIGIN: boolean`: モデルをシーンの中央に配置するかどうか。 ジオロケーションモデルに有効です。
- `USE_FAST_BOOLS: boolean`: より高速な（正確さに欠ける）ブーリアンロジックを使用するかどうか。
- `BOOL_ABORT_THRESHOLD?: number`: ブーリアン演算の計算の限界。
- `CIRCLE_SEGMENTS_LOW?: number`: 低いセグメントの曲線の解像度。
- `CIRCLE_SEGMENTS_MEDIUM?: number`: 中程度のセグメントの曲線（例：IfcSweptDiskSolid）の解像度です。
- `CIRCLE_SEGMENTS_HIGH?: number`: 高セグメントの曲線（例：IfcCircle）の解像度です。

#### 例

ファイルが地理的に配置されていて、それをシーンの原点に持ってきたい場合。

```js
await ifcLoader.ifcManager.applyWebIfcConfig({
  COORDINATE_TO_ORIGIN: true,
  USE_FAST_BOOLS: false,
});
```

### useWebworkers

```js
  async IfcLoader.IfcManager.useWebWorkers (
                                    active: boolean,
                                    path?: string
                                    ): void;
```

👷‍♀️👷‍♀️👷‍♀️
解析ロジックを含むウェブワーカーをトグルします。API の残りの部分は同じですが、すべてのロジックはマシンの別のスレッドで実行されるため、重い操作でアプリがブロックされることはありません。

🚧 ファイル`IFCWorker.js`をプロジェクトにコピーして、第 2 引数に相対パスを渡す必要があります。

#### 関数の引数

- `active`: ウェブワーカーを使用するかどうか。
- `path`: ワーカーファイルの相対パスです。`active = true`の場合に必要です。

#### 例

というフォルダに`IFCWorker.js`がある場合。`files` という名前のフォルダにあります。

```js
  await ifcLoader.ifcManager.useWebWorkers({
                                true,
                                "files/IFCWorker.js"
                                            });
```

### useJSONData

```js
  async IfcLoader.IfcManager.useJSONData (
                                    useJSON: boolean
                                    ): void;
```

🚀
WASM データの代わりに JSON プロパティデータを使用し、メモリ消費量が大幅に減少します。以下のシナリオでのみ使用してください。

- IFC のプロパティにアクセスする必要がない場合は。
- プロパティを JSON で提供する場合。
  IFC ファイルからの JSON の取得は、以下のように事前に行うことができます[この例](https://github.com/IFCjs/hello-world/tree/main/examples/web-ifc/ifc-to-json/node)。

#### 関数の引数

- `useJSON`: JSON データを使用するかどうか。

#### 例

```js
await ifcLoader.ifcManager.useJSONData(true);
```

### addModelJSONData

```js
  async IfcLoader.IfcManager.addModelJSONData (
                                modelID: number,
                                data: { [id: number]: JSONObject
                                        ): void;
```

📦📦📦
モデルのプロパティを JSON データとして追加します。Web Worker を使用している場合は、オーバーヘッドを避けるために、代わりに`loadJsonDataFromWorker()`を使用してください。

#### 関数の引数

- `modelID`: IFC モデルの ID です。
- `data`: のデータを，キーが expressID，値がプロパティであるオブジェクトとして取得します。さらに、各値はプロパティとして、`expressID`（アイテムの ID）と`type`（アイテムの IFC クラス）を持っています。

#### 例

```js
await ifcLoader.ifcManager.addModelJSONData(0, jsonData);
```

### loadJsonDataFromWorker

```js
  async IfcLoader.IfcManager.loadJsonDataFromWorker (
                                        modelID: number,
                                        path: string
                                            ): void;
```

📦📦📦
JSON ファイルから IFC モデルのデータを Web Worker から直接読み込みます。ウェブワーカーを使用していない場合は、代わりに `addModelJSONData()` を使用してください。

#### 関数の引数

- `modelID`: IFC モデルの ID です。
- `path`: JSON ファイルへのパスは、Web Worker ファイルへの相対パスです。

#### 例

```js
await ifcLoader.ifcManager.loadJsonDataFromWorker(0, "path/to/data.json");
```

## 🔍 ゲッターズ

### getExpressId

```js
  IfcLoader.IfcManager.getExpressId (
                        geometry: BufferGeometry,
                        faceIndex: number
                        ): number;
```

💳
IFC 要素の Express ID をフェースインデックスから取得する。

<IfcCard icon="📜" title="なぜ顔認証なのか？">
  なぜなら、3D空間のオブジェクトをマウスで選択すると、その面のインデックスが得られるからです。通常、その面をハイライトしたり、分離したり、すべてのプロパティを取得するためには、その面が属するオブジェクトのIDが必要です。
</IfcCard>

#### 関数の引数

- `geometry` マウスでクリックしたモデルの [ジオメトリ](https://threejs.org/docs/#api/en/core/BufferGeometry) です。

- `faceIndex` [raycaster](https://threejs.org/docs/#api/en/core/Raycaster)で交差した顔のインデックスです。レイキャスターに慣れていない方もご安心ください。ピッキングについてのチュートリアルで詳しく説明します。

#### 例

```js
const intersected = raycaster.intersectObject(mesh)[0];
const index = intersected.faceIndex;
const id = ifcLoader.ifcManager.getExpressId(mesh, index);
```

### getIfcType

```js
  IfcLoader.IfcManager.getIfcType (
                        modelID: number,
                        id: number,
                        ): string;
```

🎸🪕🎻
指定された要素の IFC タイプを取得します（例：IFCWALL）。

<IfcAlert>
  IFCの要素には、必ず関連するタイプがあります。IfcWall、IfcSlab、IfcWindow、IfcDoorなどです。
</IfcAlert>

#### 関数の引数

- `modelID` IFC モデルの ID です。

- `id` プロパティを取得したいアイテムのエクスプレス ID。これは、`getExpressId` (3D ビューでオブジェクトを選択している場合)、または `getAllItemsOfType` や `getSpatialStructure` でモデルをトラバースすることで取得できます。

#### 例

```js
const model = ifcModel.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const type = manager.getIfcType(model, id);
```

### getAllItemsOfType

```js
  async IfcLoader.IfcManager.getAllItemsOfType (
                            modelID: number,
                            type: number,
                            verbose: boolean
                            ): number[] | object[];
```

🎸🎸🎸
指定されたモデルの指定された IFC タイプのすべてのオブジェクト（すべての壁、すべての床、すべての窓など）を返します。これは、`expressID`の配列を返すか、（`verbose = true`の場合）見つかったアイテムのプロパティを含むオブジェクトの配列を返すことができます。

#### 関数の引数

- `modelID` IFC モデルの ID です。

- `type` 取得したい要素の IFC タイプです。これらのタイプは `web-ifc` から直接インポートすることができます (以下の例を参照)。

- `verbose` true の場合、見つかったすべてのアイテムのプロパティを取得します。大きなモデルでは、この処理に時間がかかることがあるので、注意が必要です。

#### 例

```js
import { IFCWALLSTANDARDCASE as W } from "web-ifc";

const manager = loader.ifcLoader.ifcManager;
const walls = await manager.getAllItemsOfType(0, W, false);
```

### getItemProperties

```js
  async IfcLoader.IfcManager.getItemProperties (
                            modelID: number,
                            id: number,
                            recursive = false
                            ): object[];
```

📕🔍
与えられた要素のネイティブ・プロパティを取得します。IFC スキーマでは、プロパティには直接プロパティと間接プロパティの 2 種類があります。間接的なプロパティ（psets、qsets、type プロパティ）の取得方法については後述します。

<IfcAlert>
  プロパティに関連するすべてのメソッドは、キーがプロパティの名前で、値がその値であるオブジェクトの配列を返します。
</IfcAlert>

#### 関数の引数

- `modelID` IFC モデルの ID です。

- `id` プロパティを取得したいアイテムのエクスプレス ID。これは、`getExpressId` (3D ビューでオブジェクトを選択している場合)、または `getAllItemsOfType` や `getSpatialStructure` でモデルをトラバースすることで取得できます。

- `recursive` true の場合、参照されているすべての要素のプロパティを再帰的に取得します。大きなモデルでは、この処理に時間がかかることがありますので、注意してください。

#### 例

```js
const model = ifcModel.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const props = await manager.getItemProperties(model, id, false);
```


### createSubset

```js
  IfcLoader.IfcManager.createSubset (
                        config: SubsetConfig
                        ): object;
```

🧱✨
新しい幾何学的サブセットを作成します。

#### 関数の引数

- `config` 以下のオプションを持つ設定オブジェクトです。

  - `scene` モデルが配置されているシーン。

  - `modelID` モデルの ID です。

  - `ids` サブセットに適合するモデルのアイテムの ID を表す。

  - `removePrevious` このモデルの前のサブセットをこの素材で削除するかどうか。

  - `material` (オプション) サブセットに適用する素材。素材が指定されていない場合、サブセットには元の素材が適用されます。

  - `customID` (オプション) サブセットに付けるオプションのカスタム名。これにより、同じ材料で複数のサブセットを作成することができます。

#### 例

```js
  const model = ifcModel.modelID;
  const manager = loader.ifcLoader.ifcManager;
  const config = {
        modelID: model
        scene: scene,
        ids: [id],
        removePrevious: true
  }
  manager.createSubset(config);
```

### removeSubset

```js
  IfcLoader.IfcManager.removeSubset (
                        modelID: number,
                        material?: Material,
                        customID?: string,
                        ): object;
```

指定された幾何学的サブセットを削除します。

#### 関数の引数

- `modelID` IFC モデルの ID です。

- `material` (オプション) サブセットに割り当てられた素材（ある場合）。

```js
const model = ifcModel.modelID;
```

### removeFromSubset

```js
  IfcLoader.IfcManager.removeFromSubset (
                        modelID: number,
                        ids: number[],
                        customID?: string,
                        material?: Material
                        ): void;
```

🧱💥🤏
指定した幾何学的サブセットから指定した項目を削除します。

#### 関数の引数

- `modelID` IFC モデルの ID です。

- `ids` サブセットから削除するアイテムの ID。

- `customID` (オプション) サブセットに付けるオプションのカスタム名。これにより、同じ材料で複数のサブセットを作成することができます。

- `material` (オプション) サブセットに適用する素材。素材が指定されていない場合、サブセットには元の素材が適用されます。

#### 例

```js
import { IFCWALLSTANDARDCASE as W } from "web-ifc";
const model = ifcModel.modelID;
const manager = loader.ifcLoader.ifcManager;
const walls = await manager.getAllItemsOfType(0, W, false);
manager.removeFromSubset(model, walls);
```
