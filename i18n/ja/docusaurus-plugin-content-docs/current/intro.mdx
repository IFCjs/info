# Introduction

IFC.js がどのようなものなのか、ここでは**5 分以内で確認できます**

## What is IFC?

設計者は、建物の設計図を一枚一枚描くのではなく、ゲーム「The SIMS」内での建物のように、建物の 3D モデルを作成するようになりました。
作成された建物モデルには形状と情報の両方が含まれていることから、この作業方法は BIM（Building Information Model）と呼ばれています。

しかし、建物は非常に複雑なので、これらのモデルをすべて作成できる単一のアプリケーションはありません。
各アプリケーションには、形状のモデリング、構造の計算、エネルギーシミュレーション、プロジェクトドキュメントの作成など、それぞれの得意とする分野があります。
これらのツールは、世界中の複数の開発者から提供されており、それぞれが独自のフォーマットで動作します。

つまり、構造計算アプリケーションが作成した構造モデルは、エネルギーシミュレーションプログラムでは読めませんし、その逆もまた然りです。
では、どうすれば複数のツールから建物のモデルを作成することができるのでしょうか。
その答えは IFC です。

IFC とは、BuildingSMART 社が作成した、建物の 3D モデルを格納するためのフォーマットです。
オープンなので、誰でも読み書きすることができます。
これは、3D やビデオゲーム用の OBJ や glTF フォーマットに相当します。
IFC のおかげで、開発者は誰でも自由に BIM アプリを作ることができ、建設分野の大手開発会社のアプリと連携することができます。
これはうれしいですよね！

## The problem of IFC

BIM アプリケーションを作ろうとする人は、他のツールから情報をインポート・エクスポートできるように IFC ファイルを読み書きする必要があります。
しかし残念ながら、IFC は読み書きが難しいフォーマットです。

何千ページものドキュメントがあり、自作の IFC パーサーを作成・維持するのは大変な作業です。
実際のところ、これは数人の開発者がフルタイムで取り組んでいる企業にしかできないことです。
これではもうそんなにオープンな感じではないですよね？

```js title="IFC ファイルの中身は以下のようになっています:"

#6699= IFCCARTESIANPOINT((0.,-1.70530256582424E-13));
#6701= IFCAXIS2PLACEMENT2D(#6699,#23);
#6702= IFCRECTANGLEPROFILEDEF(.AREA.,'SH_Curtain wall',#6701,5700.,1380.);
#6703= IFCCARTESIANPOINT((892.,-253.399999999998,150.));
#6705= IFCAXIS2PLACEMENT3D(#6703,#15,#19);
#6706= IFCEXTRUDEDAREASOLID(#6702,#6705,#19,506.8);
#6707= IFCCOLOURRGB($,0.501960784313725,0.501960784313725,0.501960784313725);

```

パーサーを 1 から作り、IFC スキーマが変更されるたびにメンテナンスする気があったとしても、
この問題に取り組むにはいくつかの自明ではない問題があります。

- 非常に大きなファイルのメモリをどのように管理するか？
- ジオメトリ生成をいかに効率的に実装するか？
- IFC が正しく定義されていない場合の対処法は？

## IFC.js to the rescue

ジオメトリやデータのインポート／エクスポートという点では全員の目的は共通しているのに、
すべての開発者がそれぞれのアプリケーションに独自の IFC ファイルのインポート／エクスポートを実装しなければならないというのは意味がありません。

**IFC.js**は、IFC ファイルの読み書きを非常に簡単に行うことができる JavaScript ライブラリです。
これにより、建築・建設アプリケーションの開発者は、IFC を容易に扱うことができ、ビジネスに本当に価値を与える機能性に集中することができます。

## IFC.js in a nutshell

IFC.js は IFC ファイルを読み込んで、2 つのものを生成します。

- ジオメトリ：IFC.js は、Three.js や Babylon.js などの 3D ライブラリと互換性があるため、3D シーンを生成することができます。
  つまり、3D の BIM ツールをすぐに作成することができます。

- データ：ジオメトリに関連するすべてのプロパティに高いレベルでアクセスできます。
  つまり、建築物の部品やその材料、熱特性、構造強度などのデータに簡単にアクセスできます。

IFC.js を使った BIM アプリケーションの作成は、JavaScript、HTML、CSS でウェブページを作成するのと同じくらい簡単です。

export const Highlight = () => (
  <iframe
    width="100%"
    height="300"
    src="https://ifcjs.github.io/hello-world/landing-page/"
    allowfullscreen="allowfullscreen"
    allowpaymentrequest
    frameborder="0"
  ></iframe>
);

<Highlight></Highlight>

## Who is this library for

このライブラリは、BIM アプリケーションを開発したいと考えている人、
つまり、建設業界向けのアプリケーションを作成している開発者と、IFC ファイルに含まれるデータを活用したいと考えている建築家や
その他の専門家の両方を対象としています。

このライブラリを使用するには、JavaScript、HTML、CSS の基本的な知識が必要となります。
この知識は、このドキュメントの範囲を超えています。
また、IFC のジオメトリを表示するために、何らかの 3D ライブラリ（Three.js または Babylon.js）を使用する必要があります。
どこから手をつけていいかわからない場合は、[こちら](https://threejsfundamentals.org/)をご覧になるか、
[Discord チャンネル](https://discord.com/invite/g7Uzn2KSwB)に参加して、気軽に質問してみてください。
