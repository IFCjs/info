🎮 IFC.jsは、ブラウザ上でIFCモデルを読み込み、表示、編集するためのJavaScriptライブラリです。あなたのIFCモデルで<a href="https://ifcjs.github.io/web-ifc-viewer/example/index">ライブデモを</a>試し、IFC.jsのパワーを発見してください。

## このライブラリは何ですか？

### IFCとは？

建築家やエンジニアは、かつて建物の設計図をひとつひとつ描いていました。しかし、今は違います。BIM（ビルディング・インフォメーション・モデリング）により、建築家とエンジニアはプロジェクトのモデルを作成するようになりました。これらのモデルには、プロジェクトを実現するために必要なジオメトリと情報の両方が含まれており、ゆりかごから墓場まで、プロジェクト全体を通してすべての情報のハブとなる必要があります。

しかし、建築物は非常に複雑であるため、これらのモデル全体を作成できる単一のアプリケーションは存在しません。各アプリケーションは、形状のモデリング、構造計算、エネルギーシミュレーション、プロジェクトドキュメントの作成など、特定のミッションを持っています。

これらのツールは、世界中の複数の開発元から提供されており、それぞれが独自のフォーマットで動作します👀。つまり、構造計算アプリケーションで作成した構造モデルは、エネルギーシミュレーションソフトでは読み込めませんし、その逆も然りです。では、どのようにして複数のツールから建物のモデルを作ることができるのでしょうか？

<a href="https://www.buildingsmart.org/standards/bsi-standards/industry-foundation-classes/"><strong>IFCとは</strong></a>、BuildingSMART社が作成した、建物の3Dモデルを格納するためのフォーマットです。オープンなフォーマットなので、誰でも読み書きできます。やったーーー

<IfcAlert>IFCのおかげで、どんな開発者でも自由にBIMアプリを作ることができ、建設分野の大手開発会社のアプリと通信することができます。</IfcAlert>

### IFCの問題点

BIMアプリケーションを作ろうとする人は、他のツールから情報をインポートしたりエクスポートしたりできるように、<strong>IFCファイルを読み書き</strong>する必要があります。残念ながら、IFCは読み書きが難しいフォーマットです。何千ページものドキュメントがあり、自作のIFCパーサーを作成・維持するのは大変な作業です。

<IfcCard title="IFCは買えるのか？">これは、何人もの開発者がフルタイムで働いている会社だけができることです。もうそんなにオープンな感じはしませんよね？</IfcCard>

IFCが内部でどのように見えるか興味がある方のために、ここに例を示します。このように何十万行もパースすることを想像してみてください。

```js
#6699= IFCCARTESIANPOINT((0.,-1.7053025E-13));
#6701= IFCAXIS2PLACEMENT2D(#6699,#23);
#6703= IFCCARTESIANPOINT((892.,-253.399999,150.));
#6705= IFCAXIS2PLACEMENT3D(#6703,#15,#19);
#6706= IFCEXTRUDEDAREASOLID(#6702,#6705,#19,506.8);
#6707= IFCCOLOURRGB($,0.50196,0.501913,0.501960);
```

💀 ゼロからパーサーを作り、IFCスキーマの変更のたびにメンテナンスすることをいとわないとしても、この問題に取り組むには、自明でないいくつかの問題があります。非常に大きなファイルのメモリをどのように管理するか？幾何学的な生成を効率的に実装するにはどうすればよいか？IFCが正しく定義されていない場合、どうすればよいか？

### IFC.jsの救済

すべての開発者が、アプリケーション用に独自のIFCファイルリーダとライタを実装しなければならないのは理にかなっていないでしょう。特に、<strong>ジオメトリとデータのインポートとエクスポート</strong>という、同じものを求めている場合はなおさらです。

<IfcCard>
  IFC.jsは、<b>IFCファイルの読み書きを</b>超簡単に行うためのJavaScriptライブラリです。
</IfcCard>

これにより、建築・建設アプリケーションの開発者は、IFCで楽に作業し、ビジネスに付加価値を与える機能に集中することができます。

## IFC.jsをひとことで言うと

IFC.jsには、2つの焦点があります。

<IfcCard icon="💎" title="ジオメトリー">
  IFC.jsは、Three.jsやBabylon.jsなどの3Dライブラリと互換性があるため、3Dシーンを生成することができます。つまり、3D BIMツールをすぐに作成することができます。
</IfcCard>

<IfcCard icon="🔗" title="データ">
  そのジオメトリに関連するすべてのプロパティへのハイレベルなアクセス。つまり、建築部材やその材料、熱特性、構造強度などに関するデータに簡単にアクセスできる。
</IfcCard>

IFC.jsを使ったBIMアプリケーションの作成は、JavaScript、HTML、CSSでWebページを作成するのと同じように簡単に行えます。

<Scene link={"https://ifcjs.github.io/hello-world/examples/web-ifc-three/introduction/"} />

```js
import { IfcLoader } from "web-ifc-three";
import { Scene } from "three";

// Creates THREE.js scene
const scene = new Scene();

// ...

// Loads IFC and adds it to the scene
const ifcLoader = new IfcLoader();
ifcLoader.load(ifcURL, (geometry) => scene.add(geometry));
```

## このライブラリは誰のためにあるのか

このライブラリは、BIMアプリケーションの開発を希望するすべての人のためのものです。これには、建設業界向けのアプリケーションを作成する開発者と、IFCファイル内のデータを活用したい建築家やその他の専門家の両方が含まれます。

このライブラリを使用するためには、JavaScript、HTML、CSSの基本的な知識が必要です。この知識は、このドキュメントの範囲を超えています。また、IFCのジオメトリを表示するために、いくつかの3Dライブラリ（Three.jsやBabylon.js）を使用する必要があります。

何から始めたらいいかわからないという方は、<a href="https://threejs.org/manual/">こちらを</a>ご覧いただくか、<a href="https://discord.gg/FXfyR4XrKT">Discordチャンネルに</a>参加してご挨拶することをお勧めします😊。

## なぜIFC.jsは違うのか？

IFC.jsは、最もユビキタスな言語の一つであるJavaScriptによって、またそのために書かれたライブラリです。つまり、ウェブブラウザ、デスクトップ、モバイルアプリケーションと互換性があります。また、JavaScriptは最も習得しやすい言語の一つであり、HTMLやCSSでユーザーインターフェースを構築することが可能です。つまり、IFC.jsを使ったBIMアプリケーションの作成は、Webページを作成するのと同じくらい簡単なのです。

### 速度

⚡️ Webアプリケーションの経験がある方なら、JavaScriptベースのIFCライブラリの弱点はパフォーマンスだと思うかもしれません。しかし、このライブラリのコアは、WebAssemblyと<a href="https://emscripten.org/">Emscriptenを</a>組み合わせたC++で開発されています。IFC.jsは、ブラウザ上で直接実行しながら、デスクトップアプリケーションの速度や性能に近づけることができます。
