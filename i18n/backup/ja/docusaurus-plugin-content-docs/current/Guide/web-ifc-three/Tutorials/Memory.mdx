## はじめに

📭📫
多くの場合、**BIM ビューアを開いたり閉じたりできるアプリケーションを作りたい**でしょう。 当たり前のように思われるかもしれませんが、問題があります。**メモリ管理** です。

🤔
さて、Web アプリケーションではメモリ管理は一般的な問題ではないので、多くの人が頭を悩ませていることでしょう。JavaScript やその他のモダンな言語には、**自動メモリ管理/ガベージコレクタ**があり、プログラマはオブジェクトや配列を解放することを心配する必要はありません。

✍🤖❓
However, when working with Three.js, data going to the graphics card (e.g. buffers) is **not affected by this automatic memory management**.

🤕
つまり、Three.js で作ったビューアを閉じると（IFC.js の有無に関わらず）、💦 **メモリリーク** が発生するのです。つまり、ユーザーの RAM メモリの一部をブロックしているのです。リークが大きくなりすぎると、アプリケーションの動作がどんどん遅くなり、**クラッシュ**する可能性があります。

<IfcAlert>
  このトピックに関する詳しい情報は
  <a href="https://threejs.org/docs/#manual/en/introduction/How-to-dispose-of-objects">
    こちら
  </a>
  でご覧いただけます。
</IfcAlert>

🚀
例えば、**React**や**Angular**などのフレームワークやライブラリを使って**SPA**（**Single Page Application**）を作成している場合、これは特に重要になります。このような場合、ウェブアプリケーションが再読み込みされることはなく、メモリリークが蓄積されていきます。

幸いなことに、IFC.js はこの点を考慮し、**使用済みメモリを非常に簡単に解放することができます**。では、その方法を見てみましょう。👇

## 方法

### メモリの監視

👀
Three.js の最適化されたシーンをあまり扱ったことがない人が最初に抱く疑問は、**どのようにしてアプリケーションがどれくらいのメモリを消費しているかを見るか**ということでしょう。そうでなければ、メモリリークを検出することは不可能です。

🌍
これにはいくつかの方法があります。1 つ目は、使用しているブラウザの**開発者向けツール**を使用することです。例えば、Google Chrome には「メモリ」という項目があり、スナップショットを撮って、現在のタブがどれだけのメモリを消費しているかを確認することができます。

🤖
しかし、より便利な方法は、ライブラリ [stats.js](https://github.com/mrdoob/stats.js/) を利用することです。このライブラリは、Three.js アプリケーションのパフォーマンスとメモリ\*\*を監視することができます。npm i stats.js` でインストールでき、以下のように使用できます。

```js
// 統計情報
const stats = new Stats();
stats.showPanel(2);
document.body.append(stats.dom);

// アニメーションのループ
const animate = () => {
  stats.begin();

  // その他をアップデート
  stats.end();
  requestAnimationFrame(animate);
};

animate();
```

📋
これにより、**実行中のシーンの統計情報**が表示されます。これらをクリックすることで、アプリのパフォーマンスとメモリ使用量を切り替えることができます。

### ディスポーザブルメモリ

✌
では、IFC.js が消費するメモリをすべて削除する関数を作ってみましょう。注意する点は 2 つあります。

- **以前に WASM ファイルへの**パスを指定した場合、web-ifc-three の新しいインスタンスで再度指定する必要があります。

- これは重要です。ロードした IFC モデルへの参照を配列、オブジェクト、またはクラスに保存した場合、**それらを手動で削除する必要があります**。さもなければ、メモリは解放されません。

```js
async function releaseMemory() {
  // これにより、すべてのIFCLoaderのメモリが解放されます。
  await ifcLoader.ifcManager.dispose();
  ifcLoader = null;
  ifcLoader = new IFCLoader();

  // 以前にwasmのパスが設定されていた場合、
  // それをリセットする必要があります
  await ifcLoader.ifcManager.setWasmPath("../../../");

  // IFCモデルが配列やオブジェクトである場合は、
  // そこでも解放する必要があります。
  // そうしないと、ガベージコレクションされません。
  models.length = 0;
}
```

### 簡単な GUI を設定する

💅
最後に、HTML ボタンを作成し、先ほど作成した関数にリンクさせます。

```html
<input type="button" id="memory-button" value="Release memory" />
```

```js
// メモリ廃棄の設定
const button = document.getElementById("memory-button");
button.addEventListener(`click`, () => releaseMemory());
```

😎 アプリケーションはこのような感じです。

<Scene
  link={

モデルを読み込んで、メモリを解放してみてください。元に戻るのがわかると思います。ブラウザのガベージコレクタには時間がかかるので、**数秒かかることがあります**ので注意してください。また、ブラウザの開発者ツールを使って、手動でガベージコレクタを適用することもできます。

### SPA

🌈
**SPA**や**Single Page Application**は、バックエンドとの通信が少なく、効率的であるため、非常に一般的なアプリケーションです。これらのアプリケーションを作るツールの例としては、**React**、**Angular**、**Vue**などがあります。

👶👴
多くの場合、**これらのツールは、アプリケーションコンポーネントのライフサイクルを自動的に管理します**。例えば、ユーザーが 3D ビューを閉じたとき、そのビューを含む HTML 要素は自動的に破棄されます。

🚧🚧🚧
メモリリークを防ぐには、3D ビューを含むコンポーネントが破壊されるたびに、**このチュートリアルで見たようにアプリケーションメモリを解放する必要があります**。多くの場合、コンポーネントが破壊されるたびに、このロジックを自動的に実行するフックまたは同様のメカニズムがあります。
